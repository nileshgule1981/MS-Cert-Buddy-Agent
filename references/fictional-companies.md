# Fictional Microsoft Companies

This document contains a list of fictional company names that Microsoft uses in documentation, training materials, and certification exams. These companies are used in scenarios to make examples feel realistic without referencing actual businesses.

**Usage Guidelines:**
- Use these fictional company names when creating practice questions, labs, or study materials
- These companies help create authentic workplace scenarios without legal or confidentiality concerns
- Most associated domains redirect to microsoft.com

---

## Commonly Used Companies for AI-102 Scenarios

The following companies are particularly well-suited for AI-102 certification scenarios:

### Contoso, Ltd.
- **Domain:** contoso.com, contoso.info
- **Description:** The most widely used Microsoft fictional company. Appears in Azure, Microsoft 365, and countless other Microsoft products.
- **Best for:** General Azure AI solutions, enterprise AI deployments, multi-service scenarios

### Fabrikam, Inc.
- **Domain:** fabrikam.com
- **Description:** Technology/manufacturing company used extensively in Microsoft documentation and demos.
- **Best for:** Computer vision scenarios (manufacturing quality control), document intelligence (invoices, forms)

### Tailwind Traders
- **Description:** Modern retail company popularized at Microsoft Ignite events.
- **Best for:** Retail AI scenarios, conversational AI, customer service bots, product recommendations

### AdventureWorks Cycles
- **Domain:** adventure-works.com
- **Description:** Bicycle manufacturer and retailer, famous from SQL Server sample databases.
- **Best for:** E-commerce AI scenarios, product search, customer support chatbots

### Northwind Traders
- **Domain:** northwindtraders.com
- **Description:** Classic import/export trading company from Microsoft Access demos.
- **Best for:** Document intelligence, invoice processing, language translation scenarios

## Best Practices for Using Fictional Companies

### For Practice Questions
1. **Match industry to scenario**: Use Fabrikam for manufacturing, Contoso Bank for finance, Fourth Coffee for small business
2. **Consistency**: If a question series involves the same company, use realistic details consistently
3. **Avoid mixing contexts**: Don't make a bank suddenly need manufacturing quality control

### For Practice Labs
1. **Use realistic business needs**: Base scenarios on actual use cases for that industry
2. **Keep it simple**: Don't over-complicate the fictional company's background
3. **Resource naming**: Use the company name in Azure resource names (e.g., `contoso-ai-rg`, `fabrikam-vision-ai`)
